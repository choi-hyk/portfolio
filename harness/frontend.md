# 프론트엔드 규칙

## 파일 구조

```text
src/app/          Next.js App Router 페이지
src/components/   공용 UI 컴포넌트
src/data/         정적 데이터 어댑터
src/i18n/         언어 설정과 사전
src/types/        프로젝트 내부 타입 선언
content/          MDX 콘텐츠
harness/          프로젝트 규칙 문서
```

## 네이밍

- React 컴포넌트 파일은 kebab-case를 사용한다: `project-card.tsx`.
- React 컴포넌트 이름은 PascalCase를 사용한다: `ProjectCard`.
- 데이터 파일은 kebab-case 또는 도메인명을 사용한다: `projects.ts`, `profile.ts`.
- 변수와 함수는 camelCase를 사용한다.
- 고정 설정값도 기본적으로 camelCase를 사용하되, 외부 생태계에서 대문자 상수를 요구하는 경우는 예외로 둔다.

## 컴포넌트

- 페이지 내부에 직접 마크업을 늘리기 전에 공용 컴포넌트를 먼저 사용한다.
- 컴포넌트는 작고 읽기 쉽게 유지한다.
- 컴포넌트 props는 `type`으로 명시하고, 암묵적인 `any`를 만들지 않는다.
- 컴포넌트별로 깊게 중첩된 스타일 규칙을 만들지 않는다.
- 반복되는 카드, 제목, 링크, 버튼, 목록 아이템은 공용 컴포넌트로 유지한다.

## 스타일링

- Tailwind utility class와 공용 레이아웃 패턴을 사용한다.
- 간격은 `gap`, `px`, `py`, `mx-auto`, `max-w-*` 같은 일관된 규칙을 우선한다.
- 명확한 레이아웃 이유가 없으면 일회성 픽셀 값을 만들지 않는다.
- 텍스트는 부모 영역을 넘치거나 주변 UI와 겹치면 안 된다.
- 카드는 단순하게 유지한다: border, 흰 배경, 작은 radius, 절제된 shadow.
- 제품상 이유가 없으면 장식용 gradient, blob, 시각적 노이즈를 추가하지 않는다.

## 목록, 페이지네이션, 스크롤

- 많은 카드 목록을 제한 없이 한 번에 렌더링하지 않는다.
- 홈 화면에는 선별된 항목만 보여준다.
- 컬렉션 페이지는 콘텐츠가 많아지면 페이지네이션, 필터, 스크롤 영역을 사용한다.
- 첫 구현은 정적 제한으로 충분하다. 실제 페이지네이션은 항목 수가 늘어난 뒤 추가한다.

## 캔버스 노드와 엣지

- 캔버스 페이지는 공용 `WorkflowCanvas`와 `CanvasNode`, `CanvasEdge` 모델을 사용한다.
- 루트 레이아웃은 공용 `PortfolioPageShell`을 한 번만 사용해 페이지 이동 중 사이드바 상태를 유지한다.
- 경력과 프로젝트 상세 페이지는 각각 `/experience/[slug]`, `/projects/[slug]` 하위 경로를 사용한다.
- 사이드바 상위 항목은 선택적인 `children`을 지원하며 하위 목록은 항상 표시한다.
- 아이콘만 있는 버튼과 축소 사이드바 링크는 공용 `Tooltip`을 사용한다.
- 툴팁은 버튼의 화면 배치를 고려해 방향을 지정하고, 뷰포트 경계에서는 반대 방향 또는 화면 안쪽으로 자동 보정한다.
- 노드의 `x`, `y`는 고정 캔버스에서 노드 좌측 상단의 백분율 좌표이고, `width`는 캔버스 너비 기준 백분율이다.
- 카드 배경이 필요 없는 강조 문구는 `appearance: "transparent"` 노드로 구성한다.
- 노드 높이는 콘텐츠에 따라 자동으로 확장하며 내부 스크롤이나 콘텐츠 말줄임을 사용하지 않는다.
- 실제 노드 높이를 측정해 엣지 접점, 포커스 중심, 미니맵에 동일하게 반영한다.
- 노드 등장 및 이전·다음 이동 순서는 `x + y`가 작은 순서로 계산해 좌측 상단에서 우측 하단 대각선 방향으로 진행한다.
- 엣지는 수동 좌표를 갖지 않고 출발 노드와 도착 노드의 `nodeId`, `side`로 구성한다.
- 엣지의 `side`는 `top`, `right`, `bottom`, `left`만 사용하며 해당 면의 중앙에 연결한다.
- 방향 간선은 2px 진한 청록 실선과 화살촉으로, 무방향 간선은 2px 청록 점선으로 표시한다.
- 노드 콘텐츠가 확장될 때 주변 노드가 겹치지 않도록 페이지 데이터의 좌표 간격을 충분히 확보한다.

## 테스트

- 기본적으로 테스트 파일을 무작정 만들지 않는다.
- 공용 동작, 데이터 변환, 라우팅 헬퍼, 회귀 가능성이 있는 로직에만 테스트를 추가한다.
- UI 변경만 있는 경우에도 기본 검증은 `npm run harness`이다.
