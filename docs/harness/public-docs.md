# 공개 문서 작성 규칙

공개 저장소에 포함되는 문서는 한국어를 기본으로 작성하고, 외부에 공개해도 문제가 없는 정보만 포함한다.

## 기본 원칙

- 공개 가능한 사실, 공개 저장소 링크, 공개 문서 링크만 사용한다.
- 개인정보와 내부 업무 정보는 작성하지 않는다.
- 확정되지 않은 정보는 문서에 넣지 않는다.
- 검토 메모, 추정, TODO 성격의 문장은 공개 문서에 남기지 않는다.
- 공개 문서는 독자가 바로 이해할 수 있는 기획서, 사용법, 규칙 문서 형태로 작성한다.

## 작성해도 되는 정보

- 이름, 공개용 이메일, GitHub, Velog
- 공개 가능한 학력과 회사명
- 공개 저장소 기반 프로젝트 설명
- 공개 문서, 배포 URL, PyPI 같은 외부 공개 링크
- 사용 기술, 역할, 공개 가능한 구현 범위

## 작성하지 않는 정보

- 생년월일, 나이, 전화번호, 상세 주소
- private 저장소 내용
- 회사 내부 프로젝트명, 고객사명, 내부 아키텍처 상세
- 비공개 API, 토큰, 환경 변수 값
- 확정되지 않은 입사일, 졸업일, 성과 수치
- “확인 필요”, “추정”, “메모” 같은 내부 검토 표현

## 포트폴리오 기획 문서 규칙

- `portfolio-content-review.md`는 공개 가능한 콘텐츠 기획서로 유지한다.
- 개인 상담 기록처럼 보이는 문장보다 사이트에 반영할 수 있는 문구를 작성한다.
- 프로젝트 설명은 공개 저장소와 공개 문서에서 확인 가능한 범위로 제한한다.
- 채용 전략, 약점 분석, 개인정보 판단 같은 민감한 내용은 공개 문서에 넣지 않는다.
